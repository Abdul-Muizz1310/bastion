# 00 — Drizzle Schema + Append-Only Grant

## Goal

Define the Postgres schema via Drizzle ORM for users, sessions, magic links, and events tables. The `events` table enforces append-only semantics at the database level (INSERT-only grant on the app role), making audit tampering impossible even if application code is compromised.

## Inputs / Outputs / Invariants

- **Input:** Drizzle schema definition in `src/lib/schema.ts`, migration SQL generated by `drizzle-kit generate`.
- **Output:** Four tables (`users`, `sessions`, `magic_links`, `events`) with correct types, constraints, indexes.
- **Invariants:**
  - `users.email` is unique.
  - `users.role` is constrained to `admin | editor | viewer`.
  - `events.id` is a monotonically increasing bigserial.
  - `events` table has INSERT-only privilege for the app role — no UPDATE, no DELETE.
  - All timestamps are `timestamptz` (with timezone).
  - `events` has indexes on `(entity_type, entity_id, created_at)`, `(created_at)`, `(service, created_at)`, `(request_id)`.

## Enumerated Test Cases

### Happy path
1. Schema exports `users`, `sessions`, `magicLinks`, `events` tables.
2. `users` table has columns: `id` (uuid PK), `email` (text unique not null), `name` (text nullable), `role` (text enum not null), `createdAt` (timestamptz default now), `deletedAt` (timestamptz nullable).
3. `sessions` table has columns: `id` (text PK), `userId` (uuid FK → users.id not null), `expiresAt` (timestamptz not null), `ip` (text nullable), `userAgent` (text nullable).
4. `magicLinks` table has columns: `token` (text PK), `email` (text not null), `expiresAt` (timestamptz not null), `usedAt` (timestamptz nullable).
5. `events` table has columns: `id` (bigserial PK), `actorId` (uuid FK nullable), `action` (text not null), `entityType` (text not null), `entityId` (text not null), `service` (text nullable), `requestId` (text nullable), `before` (jsonb nullable), `after` (jsonb nullable), `metadata` (jsonb not null default {}), `createdAt` (timestamptz default now).
6. `events` table has 4 indexes: `events_entity_idx`, `events_time_idx`, `events_service_idx`, `events_request_idx`.
7. Migration SQL generated by `drizzle-kit generate` is valid and applies cleanly to an empty database.

### Edge / failure cases
8. Inserting a user with duplicate email fails with unique constraint violation.
9. Inserting a user with role `"superadmin"` (invalid enum value) — Drizzle allows any text at ORM level, but application layer must validate via Zod before insert.
10. Inserting a session with `userId` referencing a non-existent user fails with FK violation.
11. `events.metadata` defaults to `{}` when not provided.

### Security
12. After the append-only migration runs, `UPDATE` on `events` is rejected by the database.
13. After the append-only migration runs, `DELETE` on `events` is rejected by the database.
14. `TRUNCATE` on `events` is rejected by the database.
